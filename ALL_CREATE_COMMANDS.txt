   - Generates basic README.md
   - Faster generation
   - Good for quick examples

2. With --with-docs:
   - Generates comprehensive documentation in docs/ folder
   - Extracts NatSpec comments (@title, @notice, @dev, @param)
   - Includes full contract and test code
   - Includes usage instructions
   - Better for complete documentation

3. Using npm run:
   - May need -- to pass flags: npm run create-example -- <args> --with-docs
   - Or use ts-node directly for better flag handling

4. Output Directory:
   - Change ./output/ to your desired output path
   - Example: ./my-examples/ or ../generated-examples/

CATEGORY GENERATION COMMANDS (create-fhevm-category.ts)
================================================================================

Generate projects containing multiple examples from a category.

Available Categories:
- basic (14 examples) - Fundamental FHEVM operations
- access-control (3 examples) - FHE access control patterns
- input-proofs (4 examples) - Input proof usage and lifecycle
- anti-patterns (3 examples) - Common mistakes to avoid
- openzeppelin (10 examples) - OpenZeppelin confidential contracts
- advanced (5 examples) - Complex FHEVM applications

# Generate Basic Category (9 contracts)
npm run create-category basic
# Default output: ./categories/fhevm-examples-basic
# OR with custom path:
npm run create-category basic ./output/basic-examples

# Generate Access Control Category (3 contracts)
npm run create-category access-control
# Default output: ./categories/fhevm-examples-access-control

# Generate Input Proofs Category (4 contracts)
npm run create-category input-proofs
# Default output: ./categories/fhevm-examples-input-proofs

# Generate Anti-Patterns Category (3 contracts)
npm run create-category anti-patterns
# Default output: ./categories/fhevm-examples-anti-patterns

# Generate OpenZeppelin Category (10 contracts)
npm run create-category openzeppelin
# Default output: ./categories/fhevm-examples-openzeppelin

# Generate Advanced Category (5 contracts)
npm run create-category advanced
# Default output: ./categories/fhevm-examples-advanced

Note: If you don't specify an output directory, it defaults to:
      ./categories/fhevm-examples-<category-name>
      
      Categories are generated in their own folder separate from individual examples
      which go to ./output/ directory.

BATCH GENERATION AND TEST COMMANDS (generate-all-examples-and-test.ts)
================================================================================

Generate all examples with documentation and automatically run tests.
Perfect for demos - one command instead of copy-pasting!

# Generate all examples with docs and run tests
npm run generate-all-and-test

# Generate all examples without running tests
npm run generate-all-and-test -- --skip-test

# Generate examples from specific category only
npm run generate-all-and-test -- --category basic
npm run generate-all-and-test -- --category openzeppelin
npm run generate-all-and-test -- --category advanced

Available categories:
  - basic (14 examples)
  - access-control (3 examples)
  - input-proofs (4 examples)
  - anti-patterns (3 examples)
  - openzeppelin (10 examples)
  - advanced (7 examples)

What it does:
  1. Generates all examples with --with-docs flag
  2. Automatically cd into each output folder
  3. Runs npm test in each folder (no npm install needed)
  4. Shows summary of successes/failures
